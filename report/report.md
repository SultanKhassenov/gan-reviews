## 2. Постановка задачи

### 2.1 Формальная постановка

Пусть $\mathcal{D} = \{(x_i, c_i)\}_{i=1}^{N}$ — датасет отзывов, где
$x_i$ — текст отзыва, $c_i \in \{1, \dots, K\}$ — категория товара
($K = 10$).

Через предобученный энкодер $E_\theta: x \mapsto \mathbb{R}^d$
($d = 768$, XLM-RoBERTa-base) получаем эмбеддинги $z_i = E_\theta(x_i)$.

**Задача 1 (эмбеддинговые модели):** обучить условную порождающую модель
$G_\phi: (z, c) \mapsto \tilde{z}$, такую что распределение
$P_\phi(\tilde{z} | c)$ близко к $P_{\text{data}}(z | c)$ по метрикам MMD и FD.

**Задача 2 (токен-уровень):** обучить условную модель $G_\psi: (z, c) \mapsto \tilde{x}$,
генерирующую отзывы напрямую в пространстве токенов.

## 5. Методология

### 5.1 Общий пайплайн

```
Raw reviews → XLM-R embeddings → {WGAN-GP, VAE, Text-GAN} → Metrics
```

### 5.2 Извлечение эмбеддингов

XLM-RoBERTa-base (Conneau et al., 2020), 768-dim. Используется [CLS]-токен
+ mean-pooling. Максимальная длина последовательности: 128 токенов.

### 5.3 Стандартизация

Перед подачей в модели эмбеддинги стандартизуются:
$\hat{z} = (z - \mu) / \sigma$, где $\mu, \sigma$ — поканальные статистики
тренировочного набора.

## 6. Архитектуры моделей

### 6.1 WGAN-GP

**Generator** $G: \mathbb{R}^{128} \times \{1, \dots, 10\} \to \mathbb{R}^{768}$:

> _TODO: блок-схема архитектуры, см. `models/wgan_gp.py`._

- 3 линейных слоя, ширина 512, BatchNorm + LeakyReLU(0.2)
- Conditional через label embedding (64-dim), конкатенация со входом

**Critic** $D: \mathbb{R}^{768} \times \{1, \dots, 10\} \to \mathbb{R}$:

- 3 линейных слоя, ширина 512 → 256 → 1
- LayerNorm + LeakyReLU + Dropout(0.3)
- **Важно:** BatchNorm в critic запрещён (нарушает Lipschitz-условие)

**Loss:**
$$\mathcal{L} = \mathbb{E}[D(\tilde{x})] - \mathbb{E}[D(x)] + \lambda \mathbb{E}\left[(\|\nabla_{\hat{x}} D(\hat{x})\|_2 - 1)^2\right]$$
