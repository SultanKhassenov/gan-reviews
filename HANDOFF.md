## TL;DR

Финальный проект магистранта по курсу **Generative Adversarial Networks**
(преподаватель ожидает в первую очередь image-генерацию, но допускается
исследовательская работа со статистическими метриками).

**Что строим:** сравнительное исследование трёх генеративных моделей на
эмбеддингах отзывов с Kaspi.kz:

1. **WGAN-GP** на эмбеддингах XLM-RoBERTa (768-dim) — основная модель
2. **Conditional VAE** на тех же эмбеддингах — бейзлайн для сравнения
3. **Text-GAN** с Gumbel-Softmax — экспериментальная часть (ожидаем кашу,
   это часть исследовательского нарратива)

**Главный исследовательский вопрос:** где проходит граница применимости
GAN для текста — на уровне эмбеддингов или на уровне токенов?

**Дисклеймер:** студент пишет магистерскую диссертацию по теме
"Разработка интеллектуальной системы для анализа и объективизации
отзывов с использованием методов обработки естественного языка"
(тема диссертации, не курса). Это **другой проект**, но он мотивирует
выбор Kaspi-отзывов как датасета и XLM-R как backbone.

## Правила работы для AI

### Принцип разделения кода

- **Логика** (модели, метрики, парсинг) — в `.py` файлах. Их можно импортировать,
  тестировать, использовать в нескольких местах.
- **Сценарии запуска и результаты** — в `.ipynb`. Ноутбук импортирует `.py`,
  запускает, рисует графики, сохраняет outputs.
- **НЕ дублируй** код моделей внутри ноутбуков. Если что-то нужно изменить в
  модели — правь `.py`, не ноутбук.

### Что коммитим

- ✅ Все `.py` файлы
- ✅ Все `.ipynb` с outputs (графики, числа)
- ✅ `parser/urls.yaml` (твой ввод)
- ✅ `report/report.md` и финальные `.docx`
- ✅ Маленькие примеры данных (если надо для воспроизводимости)
- ❌ `data/raw/*.jsonl` (большие, в .gitignore)
- ❌ `data/embeddings/*.npy` (большие)
- ❌ `checkpoints/*.pth` (большие)

### Запуск парсера

```bash
docker compose build parser     # один раз
docker compose run --rm parser  # каждый запуск
```

При обрыве — просто запусти ещё раз, продолжит с чекпоинта.

### Запуск моделей

Локально не получится (нужна GPU для XLM-R). Через Colab:

1. Студент пушит репо на GitHub
2. В Colab открывает любой `notebooks/0X_*.ipynb` через `File → Open notebook → GitHub`
3. Первая ячейка клонирует репо, монтирует Drive
4. Запускает остальные ячейки
5. После окончания: `File → Save a copy in GitHub` — чтобы outputs тоже попали в репо

## Принципы для AI при работе с кодом

1. **Не переписывай** существующий код просто так. Если что-то выглядит странно —
   спроси у пользователя, есть причина или это бага.
2. **Не добавляй фичи**, которые не запрошены. Курс не требует, например,
   wandb-логирования — не добавляй, если не попросили.
3. **Объясняй** что делаешь перед действием, особенно если результат повлияет
   на git-историю или большие файлы.
4. **Помни:** студент работает в свободное от диссертации время. Объём проекта
   должен укладываться в ~10 часов в неделю.
5. **Документация по-русски.** Студент пишет на русском, отчёт на русском,
   защита на русском. Код можно с английскими комментариями (короткими),
   но docstrings — на русском.
